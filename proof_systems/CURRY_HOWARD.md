# Curry-Howard / dependent type theory

## Overview

The deepest and most general proof system in common use. The key idea:

    propositions are types
    proofs are programs
    verification is type checking

A proof of proposition P is a program of type P. To verify a proof,
you run the type checker. The type checker is the verifier V in the
definition. In Lean 4, Coq, and Agda, this is exactly what happens.

## Instantiation of the Definition

    F  =  types (which encode propositions)
    P  =  terms (which encode proofs / programs)
    ⊢  =  π ⊢ φ  iff  the type checker accepts π as having type φ

## Verification Procedure

The type checker:

    given a term π and a type φ
    reduce π to normal form (beta/eta reduction)
    check that the normal form has type φ
    by structural recursion on the term

This is the kernel — in Lean 4 it is a few thousand lines of code.
It is decidable for the Calculus of Constructions (the foundation
of Coq and Lean). It terminates because well-typed terms always
normalize.

## The Alphabet

Term constructors (finitely many):

    λ x => t        function abstraction
    f a             application
    (x : A) → B     dependent function type
    Type u           universe
    ...plus inductive type constructors (finitely many per definition)

The whole infinite space of proofs is generated by finite combination
of these constructors. Classic instance of infinite combinations
from finite primitives.

## Why This Is the Cleanest Instance

Of all proof systems, Curry-Howard most clearly instantiates the
definition because:

    1. The verifier (type checker) is explicitly implemented as code
    2. The alphabet (term constructors) is explicitly enumerated
    3. The proof objects (terms) are explicitly finite trees
    4. The decidability is a theorem (strong normalization)

There is no gap between the abstract definition and the implementation.
The definition IS the implementation.

## Meets the Definition?

Yes — most cleanly of all. Type checking is decidable. Alphabet is
a finite set of term constructors. Proof objects are finite terms.
The correspondence is exact.

## Connection to This Project

Lean 4 is a Curry-Howard system. The proofs in universal-lean,
transformer-bp-lean, and hard-bp-lean are all terms in the
Calculus of Constructions. Their verification is type checking.
The finite alphabet of Lean 4's kernel is exactly the finite
alphabet this repo claims is necessary.
